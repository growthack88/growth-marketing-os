# Content Repurposing Matrix Prompt

> Turn one flagship piece into a month of platform-native distribution — adapted per platform's consumption pattern, not chopped into clips — with scheduling order and the bilingual tier call included.

## ⚡ What it does

"Repurposing" usually means cutting a video into random clips and pasting the same caption everywhere — which is why it usually flops. Real repurposing re-expresses the core insight in each platform's native shape: the contrarian claim becomes a text post, the framework becomes a carousel, the story becomes a thread, the demo becomes a short. This prompt builds that matrix from one input, ordered so each piece feeds the next instead of cannibalizing it.

## 🎯 When I use it (real scenario)

This is the distribution layer of my own content operation — long-form flagship (YouTube/guide) at the top, everything else derived. The system exists because the math is unforgiving: a flagship piece takes days to make, and without a repurposing pass it gets one day of reach. The matrix version of the habit came from noticing which derivatives actually worked: never the auto-clips — always the pieces rebuilt for the platform, which is exactly the step this prompt refuses to skip.

## 📋 The Prompt

```
You are a content distribution strategist. Build a repurposing matrix from my flagship piece. ADAPT per platform — never copy-paste the same text twice.

THE FLAGSHIP:
[paste the script/article/guide — or its detailed outline + key moments]

CONTEXT:
- Core insight in one sentence: [X]
- My platforms (only ones I actually maintain): [LinkedIn / X / IG-TikTok shorts / newsletter / blog]
- Language strategy: [EN / AR / bilingual — if bilingual: which platforms get which language]
- Goal: [audience growth / launch traffic to LINK / authority]

BUILD THE MATRIX — extract from the flagship:
1. THE CLAIMS (2-3 contrarian or strong statements) → standalone text posts
2. THE FRAMEWORK (steps/model, if any) → carousel or numbered thread
3. THE STORY/EXAMPLE → narrative post or thread (first-person, ends with the lesson)
4. THE PROOF MOMENTS (numbers, before/after, demos) → short-form video concepts + a proof post
5. THE OBJECTION it answers → a "you're probably thinking X" post
6. THE ONE-LINERS (quotable lines as written) → X posts / story graphics

FOR EACH DERIVATIVE OUTPUT:
| # | Platform | Format | The piece (drafted, platform-native length) | Hook line | CTA (most = "none"; only 2-3 pieces point to the flagship) | Day (suggested order) |

RULES:
- Sequence over 3-4 weeks: claims early (start arguments), framework mid (saves),
  story + proof late (payoff), ONE direct flagship-promo piece per week max.
- Arabic pieces: rebuild in عامية per the platform, never translate the EN derivative.
- Each derivative must stand alone — a reader who never sees the flagship still gets value.
- Flag which 2 derivatives are strongest as PAID ad candidates.
```

## 🧠 Pro tips

- Extract the matrix BEFORE publishing the flagship — knowing the derivatives sharpens the flagship's structure (you write quotable lines on purpose).
- The claims posts are your testing lab: whichever claim gets the most argument becomes your next flagship topic. The loop compounds.
- Don't distribute to platforms you won't answer comments on — repurposing multiplies reach, and reach without reply is a brand running on autopilot.
- Bilingual: give each platform ONE language identity (per the [Bilingual Content System](../../frameworks/bilingual-content-system.md)) instead of alternating — audiences follow for a register, not just a topic.
- Feed the winning hooks back into the [Bilingual Hook Bank](../../swipe-files/hooks/bilingual-hook-bank.md) — your swipe file should grow from your own scoreboard.
